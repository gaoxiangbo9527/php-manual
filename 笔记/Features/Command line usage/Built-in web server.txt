警告
    设计这个 web 服务器的目的是帮助应用程序开发。它对于测试目的或在受控的环境中运行应用程序的演示也是十分有用的。它没打算成为一个功能齐全的 web 服
    务器。它不应该在公共的网络上使用。

从 PHP 5.4.0 起，CLI SAPI 提供了一个内置的 web 服务器。
该 web 服务器只运行了一个单线程的进程，因此，如果请求被阻塞，PHP 应用程序将停止运行。
URI 请求会从启动 PHP 的当前工作目录中提供，除非使用 -t 选项指定了显式的文档根目录。如果某个 URI 请求没有指定一个文件，则返回给定目录中的
index.php 或 index.html 。如果两个文件都不存在，则对 index.php 和 index.html 的查找会在父目录中继续进行，依次类推，直到找到一个文件或者到达
文档根目录。如果找到某个 index.php 或 index.html，则会返回它并且将 $_SERVER['PATH_INFO'] 设置为 URI 的结尾部分。否则的话，会返回一个 404
响应代码。

如果在启动 web 服务器时在命令行上给出一个 PHP 文件，它将被当作一个“路由”脚本对待。该脚本会在每个 HTTP 请求开始时运行。如果这个脚本返回了 FALSE，
那么该请求资源会按照原来的样子被返回（个人补充，返回 FALSE 表示不路由，该路由脚本就像不存在一样）。否则该脚本的的输出会被返回给浏览器。

对于扩展名为：.3gp，.apk，.avi，.bmp，.css，.csv，.doc，.docx，.flac，.gif，.gz，.gzip，.htm，.html，.ics，.jpe，.jpeg，.jpg，.js，
.kml，.kmz，.m4a，.mov，.mp3，.mp4，.mpeg，.mpg，.odp，.ods，.odt，.oga，.ogg，.ogv，.pdf，.pdf，.png，.pps，.pptx，.qt，.svg，
.swf，.tar，.text，.tif，.txt，.wav，.webm，.wmv，.xls，.xlsx，.xml，.xsl，.xsd 和 .zip 的文件，会返回标准的 MIME 类型。

来自示例
    内置的 Web 服务器的 SAPI 名称是 cli-server 。

    你可以使 web 服务器在端口 8000 上让任何接口都可以访问（比如从远程机器上访问）：
        $ php -S 0.0.0.0:8000

个人补充
    通过这种方式运行的服务，好像已经自动做好 HOST 绑定了
