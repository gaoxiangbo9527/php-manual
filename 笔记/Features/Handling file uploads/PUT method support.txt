PHP 提供了对 HTTP PUT 方法的支持，一些客户端使用该方法往服务器中存放文件。

相比使用 POST 请求上传文件来说 PUT 请求更加的简单。它看起来像这样：
    PUT /path/filename.html HTTP/1.1

这通常意味着远程的客户端希望将内容存放为 web 树中的 /path/filename.html 。然而对于 Apache 和 PHP 来说，让任何人可以自动的覆盖 web 树中的任意
文件显然不是什么好主意。因此，想要处理这样的一个请求，首先你需要告诉你的服务器你想让一个特定的 PHP 脚本来处理这样的请求。

在 Apache 中可以用 Script 指令来达到该目的。该指令几乎可以放到 Apache 配置文件中的任何位置。一个常见的位置是 <Directory> 块中，或者也可能是
<VirtualHost> 块中。可以用如下一行来完成该设置：
    Script PUT /put.php
这告诉 Apache 对于所有 URIs 匹配了放置这一行的上下文的 PUT 请求都将发送给 put.php 脚本。当然，这是建立在 PHP 启用了对 .php 扩展名的支持并且
 PHP 已经激活的假设之上。
发送给该脚本的 PUT 请求的目标资源都是脚本本身，上传的文件不应该有文件名。
