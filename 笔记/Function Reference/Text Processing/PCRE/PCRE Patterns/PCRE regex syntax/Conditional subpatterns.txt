根据一个断言的结果，或者前面的某个捕获子模式是否匹配，可以使匹配过程有条件地遵从一个子模式，或者在两个替代子模式之间进行选择。条件子模式的两种
可能形式是：

(?(condition)yes-pattern)
(?(condition)yes-pattern|no-pattern)

如果满足条件，会使用 yes-pattern；否则会使用 no-pattern（如果出现的话）。如果子模式中的替代选择超过两个，会发生一个编译时错误。

有两种类型的条件。如果圆括号之间的文本由一串数字组成，那么如果该号码的捕获子模式之前已经匹配，则满足条件。考虑下面的模式，它包含了不重要的空白
以使其更可读（假设设置了 PCRE_EXTENDED）并将它分成了三个部分，以方便讨论：

( \( )?    [^()]+    (?(1) \) )

第一部分会匹配一个可选的开始圆括号，并且如果该字符出现的话，就把它设置为第一个捕获子字符串。第二部分会匹配一个或多个不是圆括号的字符。第三部分
是一个条件子模式，它测试第一组括号（第一个子模式）是否已经匹配。如果时，即如果目标以一个开始圆括号开头，条件就为 TRUE，这样就会执行
yes-pattern 并且会需要一个关闭圆括号。否则，因为没有出现 no-pattern，该子模式就什么也不会匹配。换句话说，此模式匹配一个非圆括号的序列，可
选的被圆括号包围。

如果条件是字符串 (R)，那么如果对该模式或子模式进行了递归调用，则满足条件。在“顶层”，条件为 FALSE。

如果条件不是一个数字序列或 (R)，那么它必须是一个断言。它可以是一个肯定或否定的向前查找或向后查找断言。考虑这个模式，再次包含不重要的空白，并且
在第二行上有两个替代选择：

(?(?=[^a-z]*[a-z])
\d{2}-[a-z]{3}-\d{2}  |  \d{2}-\d{2}-\d{2} )

该条件是一个肯定的向前查找断言，它会匹配一个可选的非字母序列后跟一个字母。换句话说，它会测试主题中是否存在至少一个字母。如果找到一个字母，该主
题会与第一种替代选择相匹配；否则就会与第二种替代选择相匹配。该模式会匹配以下两种形式字符串中的一种：dd-aaa-dd 或 dd-dd-dd，其中 aaa 是字母
且 dd 是数字。
