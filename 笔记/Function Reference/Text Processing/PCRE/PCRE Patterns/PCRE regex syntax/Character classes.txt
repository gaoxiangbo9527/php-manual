一个开始方括号引入了一个字符类，该字符类被一个关闭方括号终止。独自的一个关闭方括号不是特殊的。如果需要一个关闭方括号作为类的一个成员，它应该是
该类中的第一个数据字符（位于最初的音调符号之后，如果有出现的话）或者被反斜杠转义。

字符类会匹配主题中的单个字符；该字符必须在该类所定义的字符集合中，除非该类中的第一个字符是音调符号，这种情况下，该主题字符必须不在该类所定义的
字符集合中。如果实际需要一个音调符号作为类的一员，确保它不是第一个字符，或者使用一个反斜杠转义它。
个人补充：把音调符号放在字符类的最后是一个不错的选择，如果该字符类还需要一个字面的减号，那么推荐把音调符号放在倒数第二位。

例如，字符类 [aeiou] 会匹配任何小写的元音字符，而 [^aeiou] 会匹配任何不是小写元音字符的字符。注意，音调符号只不过是“通过列举不属于类的字符
来指定属于类的字符”的一种方便的记法。它不是一个断言：它仍然消耗主题字符串中的一个字符，并且如果当前指针在字符串的末尾时会失败。

当设置了大小写不敏感（不区分大小写）的匹配时，字符类中的任意字符都同时代表了它们的大写和小写版本，举个例子，一个不敏感的 [aeiou] 会匹配 "A"
和 "a"，并且一个不敏感的 [^aeiou] 不会匹配 "A"，而一个敏感（区分大小写）的版本会匹配。

不管 PCRE_DOTALL 或 PCRE_MULTILINE 选项的设置为何，字符类中的换行符不会被以任何特殊的方式处理。一个如 [^a] 的字符类总是会匹配一个换行符。

在字符类中，减号（连字符）可以被用于去指定一个字符区间。例如，[d-m] 会匹配任何位于 d 和 m 之间的字母，包含首尾。如果字符类中需要一个减号，
那么它必须被一个反斜杠转义或者出现在一个不会被解释为指示一个区间的位置，通常作为字符类中的第一个或最后一个字符。

让字面字符 "]" 作为一个区间的结束字符是不可能的。一个如 [W-]46] 的模式会被解释为一个具有两个字符（"W" 和 "-"）的字符类后面跟着一个字面字符
串 "46]"，因此它将匹配 "W46]" 或者 "-46]"。然而，如果 "]" 被一个反斜杠转义，那么它会被解释为区间的结束，因此 [W-\]46] 会被解释为单个字符
类，其中包含一个区间且后跟两个单独的字符。也可以使用 "]" 的八进制或十六机制表示去结束一个区间。
个人补充：结合第一段和上面这段中对结束方括号的描述，对于字符类中出现的所有结束方括号，不论是字面结束方括号，还是区间结束方括号，建议通通使用反
斜杠转义，不使用调整位置或者八进制或十六机制表示形式。

区间操作以 ASCII 排序序列。它们还可以用于数值化指定的字符，例如 [\000-\037]。如果使用一个包含字母的区间，当匹配设置为大小写不敏感（不区分
大小写）时，它会匹配这两种情况下的字母。例如，[W-c] 等价于 [][\^_`wxyzabc]，并且，如果使用了 ”fr”（法国）的地域设置字符表时，[\xc8-xcb]
将会在所有模式下匹配重音 E 字符。

字符类型 \d，\D，\s，\S，\w 和 \W 也可以出现在一个字符类中，并且添加它们所匹配的字符到该字符类中。例如，[\dABCDEF] 会匹配任何十六进制数字。
音调符号可以与大写的字符类型（即 \D，\S 和 \W）一起使用，以方便地指定一组比对应的小写字符类型（即 \d，\s 和 \w）更受限制的字符。比如，字符类
[^\W_] 会匹配任何字母和数字，但不会匹配下划线。

除了 \、-、^（开始时）和终止符 ] 之外，所有非字母数字字符在字符类中都不是特殊的，但如果对它们进行转义，也不会造成损害。模式结束符总是特殊的，
在表达式中使用时必须转义。

Perl 为字符类支持 POSIX 记号法。这种记号法在方括号内使用由 [: 和 :] 括起来的名字。PCRE 也支持这种记号法。例如，[01[:alpha:]%] 匹配 "0"
，"1"，任何字母字符，或者 "%"。受支持的类名是：

字符类
    alnum	字母和数字
    alpha	字母
    ascii	码点为 0 - 127 的字符
    blank	空格和制表符
    cntrl	控制字符
    digit	十进制数字（与 \d 相同）
    graph	可打印字符，空格除外
    lower	小写字母
    print	可打印字符，包括空格
    punct	可打印字符，字母和数字除外
    space	空白（和 \s 不太一样）
    upper	大写字母
    word	单词字符（与 \w 相同）
    xdigit	十六进制数字

space 字符是 HT (9)，LF (10)，VT (11)，FF (12)，CR (13) 和 空格 (32)。注意，这个列表包括 VT 字符（代码为 11）。这使得 "space" 与
\s 不同，\s 不包含 VT（Perl 兼容）。名称 word 是 Perl 扩展的，并且从 Perl 5.8 开始 blank 是一个 GNU 扩展。另一项 Perl 扩展是否定，它
由冒号后的 ^ 字符指定。例如，[12[:^digit:]] 会匹配 "1"，"2" 或任何非数字字符。

在 UTF-8 模式中，值大于 128 的字符不会匹配任何 POSIX 字符类。从 PHP 5.3.0 和 libpcre 8.10 开始，一些字符类被更改为使用 Unicode 字符
属性，在这种情况下，上述限制不适用。详情请参考 PCRE(3) 手册。

Unicode 字符属性可以出现在字符类中。它们不能是区间的一部分。一个 Unicode 字符类后面的减号（连字符）将会按字面匹配。尝试使用一个 Unicode 字
符属性终止一个区间将会导致一个警告。
