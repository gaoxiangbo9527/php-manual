在字符类外，在默认的匹配模式下，音调符号（^）是一个断言，仅当当前的匹配点位于主题字符串的开头时，该断言才为真。在字符类内，音调（^）拥有着完全
不同的含义（见下文）。

如果涉及到许多替代选择，音调（^）不需要是模式的第一个字符，但它应该是它所出现于的每个替代选择中首先出现的东西，如果这个模式要匹配该分支的话。
如果所有可能的替代选择都以一个音调（^）开头，也就是说，如果模式被约束为只匹配主题的开头，这被称为是一个“锚定”模式。（还有其他一些构造可以引发
一个模式被锚定。）

美元符号（$）是一个断言，仅当当前的匹配点位于主题字符串的结尾或者刚好在是字符串中的最后一个字符的换行符前面时（默认），该断言才为真。如果涉及
到许多替代选择，美元（$）不需要是模式的最后一个字符，但它应该是它所出现于的任何分支中的最后一项。美元符号在字符类内没有特殊含义。

通过在编译时或者匹配时设置 PCRE_DOLLAR_ENDONLY 选项，可以改变美元符号的含义，从而让它只会匹配字符串的结束。这不影响 \Z 断言。

如果设置了 PCRE_MULTILINE 选项，音调符号和美元符号的含义会发生改变。在这种情况下，它们除了会在主题字符串的开始和结束匹配之外，还分别在内部的
"\n" 字符之后和之前立即匹配。例如，模式 /^abc$/ 在多行模式下会匹配主题字符串 "def\nabc"，但在非多行模式下则不会匹配该主题字符串。结果，在
单行模式下锚定的模式（因为所有分支都以 "^" 开头）在多行模式下不锚定。如果设置了 PCRE_MULTILINE，则会忽略 PCRE_DOLLAR_ENDONLY 选项。

注意，在两种模式下，序列 \A，\Z 和 \z 都可以被用于去匹配主题的开始和结束，并且如果一个模式的所有分支都已 \A 开头，它就是锚定的，不论是否设置
了 PCRE_MULTILINE。

个人补充：
    在默认的单行模式下，\A，\Z 和 \z 分别等价于 ^，$ 和开启 PCRE_DOLLAR_ENDONLY 的 $，所以此时没必要使用 \A，\Z 和 \z。
    多行模式才是 \A，\Z 和 \z 和用武之地。
