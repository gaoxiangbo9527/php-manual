描述
    flush ( ) : void

    冲刷 PHP 和 PHP 正在使用的后端（CGI，一个 web 服务器，等等）的系统写缓冲区。这尝试将当前输出一直推送到浏览器，但有一些注意事项。

    flush() 可能无法覆盖 web 服务器的缓冲方案，并且它对浏览器中的任何客户端缓冲没有影响。它也不会影响 PHP 的用户空间输出缓冲机制。这意味着
    你必须同时调用 ob_flush() 和 flush() 来冲刷输出缓冲区，如果你正在使用它们。

    一些服务器，尤其是 Win32 上的服务器，将仍然缓冲脚本的输出，直到将结果传输到浏览器前的缓冲终止。

    一些用于 Apache 的服务器模块，例如 mod_gzip，可能会自行进行缓冲，这将导致 flush() 不会立即将数据发送到客户端。

    甚至浏览器在显示输入之前也会缓冲它的输入。例如，Netscape 会缓冲文本直到它接收一个行结束或者一个开始标签，并且它不会渲染表格直到看到最外
    层的 </table> 标签。

    某些版本的微软 IE 将只会在接收到 256 个字节的输出后才会开始显示页面，因此你可能需要在冲刷之前发送额外的空白，以便让这些浏览器显示页面。

返回值
    没有值被返回。
