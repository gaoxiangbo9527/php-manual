描述
    set_time_limit ( int $seconds ) : bool

    设置一个脚本允许去执行的描述。如果到达了此限制，该脚本会返回一个致命的错误。默认的限制是 30 秒，或者如果存在，则为 php.ini 中定义的
    max_execution_time 值。

    当被调用时，set_time_limit() 将从零开始重启超时计数器。换句话说，如果超时时间是默认的 30 秒，并且在脚本执行的 25 秒处进行了一个如
    set_time_limit(20) 这样的调用，那么该脚本在超时之前将总共运行 45 秒。

    个人补充：如上所述，如果想自定义某个脚本的最大执行时间，就应该在该脚本的顶部调用 set_time_limit() 函数。

参数
    seconds
        最大执行时间，单位为秒。如果设置为 0，则没有时间限制。

返回值
    成功时返回 TRUE，失败时返回 FALSE 。

注意事项：
    警告 当 PHP 运行在 安全模式 下时，这个函数不会有效果。除了关闭安全模式或更改 php.ini 中的时间限制之外，没有其他变通的办法。

    注意：set_time_limit() 函数和 max_execution_time 配置指令只会影响脚本自身的执行时间。在确定脚本已经运行的最大时间时，任何花费在脚本
    执行以外的活动上的时间都不会被算进来，例如使用 system() 的系统调用、流操作、数据库查询等等。在 Windows 上不是这样的，被测量的时间是真
    实的。
