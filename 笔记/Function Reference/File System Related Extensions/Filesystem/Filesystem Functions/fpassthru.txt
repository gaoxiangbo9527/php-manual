描述
    fpassthru ( resource $handle ) : int

    在给定的文件指针上从当前位置读到 EOF 并且把结果写到输出缓冲。

    如果你已经向文件中写入了数据，那么你可能需要调用 rewind() 去把文件指针重置到文件的开头。

    如果你只是想将一个文件的内容转储到输出缓冲，而不先修改它或寻址到一个特定的偏移量，你可能会想使用 readfile()，它可以让你不用去调用
    fopen() 。

参数
    handle
        文件指针必须是有效的，并且必须指向一个由 fopen() 或 fsockopen() 成功打开（并且还没被 fclose() 关闭）的文件。

返回值
    如果发生一个错误，fpassthru() 返回 FALSE 。否则，fpassthru() 返回从 handle 读取并传递到输出的字符数。

注意事项
    注意：
        当在 Windows 系统上对一个二进制文件使用 fpassthru() 时，应该确保以二进制模式打开该文件，通过在对 fopen() 的调用所使用的模式中追
        加一个 b 。

        建议在处理二进制文件时使用 b 标志，即使你的系统不需要它，这样你的脚本将更容易移植。
