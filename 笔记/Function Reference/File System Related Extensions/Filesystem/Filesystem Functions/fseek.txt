描述
    fseek ( resource $handle , int $offset [, int $whence = SEEK_SET ] ) : int

    为由 handle 所引用的文件设置文件位置指示器。新的位置，从文件的开头按字节进行度量，是通过向 whence 所指定的位置加上 offset 得到的。

    一般来说，可以在文件结束后进行寻址；如果接下来数据被写入，那么读取文件结束和寻址位置之间的任何未被写入的区域将产生值为 0 的字节。然而，特
    定的流可能不支持此行为，特别是当它们具有底层固定大小的存储时。

参数
    handle
        一个通常使用 fopen() 创建的文件系统指针 资源。
    offset
        偏移量。

        要想移动到文件结束之前的某个位置，需要在偏移量中传递一个负值，并将其设置为 SEEK_END 。
    whence
        whence 的值有：
            SEEK_SET - 设置位置等于 offset 字节。
            SEEK_CUR - 设置位置到当前位置加上 offset 。
            SEEK_END - 设置位置到文件结束加上 offset 。

返回值
    成功返回 0；否则,返回 -1。

注意事项
    注意：如果你以追加模式（a 或 a+）打开文件，那么写入文件的任何数据总是会被追加，而不管文件位置如何，并且调用 fseek() 的结果将是未定义的。

    注意：并不是所有的流都支持寻址。对于那些不支持寻址的流，从当前位置向前寻址是通过读取和丢弃数据来完成的，其他（另外两种）形式的寻址将会失败。
