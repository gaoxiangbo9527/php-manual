描述
    session_gc ( void ) : int

    session_gc() 用于执行会话数据 GC（垃圾回收）。PHP 在默认情况下执行基于概率的会话 GC。

    基于概率的 GC 可以工作，但它有几个问题。1）低流量网站的会话数据可能不会在偏好的持续时间内被删除。2）高流量网站的 GC 可能会太频繁。3）GC
    是在用户的请求上执行的，所以用户将会遇到 GC 延迟。

    因此，对于生产系统推荐定期的执行 GC，例如，对类 Unix 系统使用 "cron"。确保通过把 session.gc_probability 设置为 0 来禁用基于概率的
    GC。

返回值
    成功时 session_gc() 返回被删除的会话数据的数量，失败时返回 FALSE。

    旧的保存处理程序不返回删除的会话数据的数量，而只会返回成功/失败标志。如果是这种情况，则删除的会话数据数变为 1，而与实际删除的数据无关。

来自示例：
    如果使用 cron 这种定时任务管理器，应该以和 php-fpm 进程或 web 服务器进程相同的用户去执行调用了 session_gc() 的脚本。
