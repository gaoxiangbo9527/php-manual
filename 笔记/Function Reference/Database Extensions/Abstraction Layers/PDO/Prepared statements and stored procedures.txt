许多更成熟的数据库支持预备语句的概念。它们是什么？它们可以被认为是一个应用程序想要运行的 SQL 的一种编译好的、可以使用可变参数自定义的模板。
预备语句提供了两个主要的好处：
    查询只需要被解析（或者说准备）一次，但却可以使用相同或者不同的参数去执行多次。当查询准备好后，数据库将分析、编译并优化执行查询的计划。对
    于复杂的查询，这个过程可能会占用相当多的时间，如果需要使用不同的参数多次重复相同的查询，则会显著降低应用程序的速度。通过使用一条预备语句，
    应用程序就可以避免重复分析/编译/优化的循环。这意味着预备语句使用更少的资源而且因此运行的更快。

    预备语句的参数不需要用引号括起来；驱动程序会自动的进行这个处理。如果应用程序只使用准备好的语句，开发人员可以确保不会发生 SQL 注入（但是，
    如果查询的其他部分是用未转义的输入构建的，那么 SQL 注入仍然是可能的）。

预备语句是如此有用，以至于它是 PDO 会为那些不支持它的驱动去模拟的唯一的一个特性。这确保一个应用程序能够使用相同的数据访问范式，无论数据库的功
能如何。
