此扩展通过验证或清理数据来过滤数据。当数据源包含未知（或外部）数据时，这尤其的有用。例如，此数据可能来自于一个 HTML 表单。

有两种主要的过滤类型：验证和清理。

验证 用于验证或检查数据是否满足特定的资格。例如，传递 FILTER_VALIDATE_EMAIL 将判断数据是否是一个有效的电子邮件地址，但不会改变数据本身。

清理 将清理数据，所以它可能会通过移除不期望的字符去修改数据。例如，传递 FILTER_SANITIZE_EMAIL 将删除不适合一个电子邮件地址去包含的字符。
那就是说，它不会验证数据。

标记是可选的，它们可与验证和清理一起使用，以根据需要调整行为。例如，当在过滤一个 URL 时传递 FILTER_FLAG_PATH_REQUIRED 将要求出现一个路径
（像 http://example.org/foo 中的 /foo）。
