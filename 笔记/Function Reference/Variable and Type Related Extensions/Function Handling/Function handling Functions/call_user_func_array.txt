描述
    call_user_func_array ( callable $callback , array $param_arr ) : mixed

    调用由第一个参数指定的 callback，而且将 param_arr 中的元素作为该回调的参数。

参数
    callback
        将被调用的回调。
    param_arr
        将会传递给该回调的参数，以一个索引数组的形式。

返回值
    成功时返回回调的返回值，发生错误时返回 FALSE 。

更新日志
    版本     描述
    5.3.0   像 parent 和 self 这样的面向对象关键字的解释已经改变了。以前，使用双冒号语法调用它们会发出 E_STRICT 警告，因为它们被解释为
            静态的。

注意事项
    注意：
        在 PHP 5.4 之前，param_arr 中的变量是按引用传递给函数的，不管函数是否期望按引用传递相应的参数。
        调用时按引用传递的形式被弃用了，但在本情况中并不会产生一条弃用通知，并且在 PHP 5.4 中 调用时按引用传递的形式 被移除了。
        进一步说，这种传递行为也不会应用于内部函数，内部函数的签名是受到尊重的。
        当函数期待一个参数是按引用传递时，按值传递会导致一个警告并且 call_user_func() 会返回 FALSE 。

    注意：使用诸如 call_user_func() 和 call_user_func_array() 函数注册的回调不会被调用，如果在前一个回调中有一个未被捕获的异常的话。
