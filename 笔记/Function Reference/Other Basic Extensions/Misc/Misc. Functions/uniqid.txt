描述
    uniqid ([ string $prefix = "" [, bool $more_entropy = FALSE ]] ) : string

    基于以微秒为单位的当前时间去获取一个带前缀的惟一标识符。

    小心 此函数不会生成密码学安全的值，并且不应该用于加密目的。如果你需要一个密码学安全的值，考虑使用 random_int()，random_bytes() 或
    openssl_random_pseudo_bytes() 。

    警告 此函数不保证返回值的惟一性。由于大多数系统通过 NTP 或类似的方式来调整系统时钟，因此系统时间是不断变化的。因此，这个函数对于进程/线
    程可能不会返回唯一的 ID。使用 more_entropy 来增加唯一性的可能性。

参数
    prefix
        例如，如果你在可能在同一微秒生成标识符的几台主机上同时生成标识符，则可能非常有用。

        使用一个空前缀，返回的字符串将是 13 个字符长。如果 more_entropy 是 TRUE，那么它将是 23 个字符。
    more_entropy
        如果设置为 TRUE，uniqid() 将会在返回值的尾部添加额外的熵（使用组合线性同余生成器），这增加了结果唯一性的可能性。

返回值
    返回一个基于时间戳的唯一标识符字符串。

    警告 这个函数尝试创建唯一的标识符，但是它不能保证返回值 100% 的唯一。

注意事项
    注意：在 Cygwin 下，必须把 more_entropy 设置为 TRUE 才能让这个函数工作。
