PHP 是一种强大的语言和解释器，不论是作为一个模块被包含到一个 web 服务器中，还是作为一个分离的 CGI 二进制程序被执行，都能在服务器上访问文件，
执行命令和打开网络连接。默认情况下，这些属性使在 web 服务器上运行的任何东西都是不安全的。

PHP 被专门设计为一种对于编写 CGI 程序来说比 Perl 或 C 更安全的语言，并且通过正确选择编译时和运行时配置选项以及适当的编程实践，它可以为你提
供所需的自由和安全性的精确组合。

由于使用 PHP 有许多不同的方法，因此有许多配置选项可以控制它的行为。大量的选项保证你可以将 PHP 用于许多目的，但也意味着这些选项和服务器配置的
组合会导致一个不安全的设置。

PHP 的配置灵活性与代码灵活性不相上下。PHP 可以被用于去构建具有某个 shell 用户所有功能的完整的服务器应用程序，或者在一个严格控制的环境中，它
可以几乎没有风险的被用于简单的服务器端包含。如何构建该环境以及它的安全性会如何，在很大程度上取决于 PHP 开发人员。

本章以一些一般性的安全建议开始，解释了不同的配置选项组合以及可以安全使用它们的情境，并描述了对于不同级别的安全性在编码时有哪些不同考虑事项。
