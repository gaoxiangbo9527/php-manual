如今，因为数据库可以让网站能够提供各种动态内容，所以它成为了任何基于 web 的应用程序的基本组件。由于非常敏感或机密的信息可以存储在数据库中，你
应该强烈考虑保护你的数据库。

为了检索或存储任何信息，你需要连接到数据库，发送一个合法的查询，获取结果，并且关闭连接。目前，在这种交互中，常用的查询语言是结构化查询语言
（Structured query language，SQL）。了解攻击者怎样能 篡改一个 SQL 查询。

正如你所猜测的那样，PHP 本身无法保护数据库。后续的章节旨在介绍如何在 PHP 脚本中访问和操作数据库的基本知识。

牢记这个简单的规则：纵深防御。你所采取的增强对数据库保护的措施越多，攻击者成功的暴露或滥用任何被存储的信息的可能性就越低。良好设计的数据库结构
和应用程序可以解决掉你最大的担忧。
