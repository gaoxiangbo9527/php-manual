在一对开始和结束标签以外的一切事物都会被 PHP 解析器忽略，这允许 PHP 文件拥有混合的内容。这允许 PHP 被嵌入到 HTML 文档中，例如去创建模板。

当 PHP 解析器碰到 ?> 结束标签时，它会简单地开始输出它发现的内容（除了紧接着的换行符）直到碰到下一个开始标签；除非是处于一个条件语句的中间，
此时 PHP 解析器会在决定跳过哪些内容之前先去判断条件语句的结果。

对于输出大段文本的情况，跳出 PHP 解析模式通常比将文本通过 echo 或 print 输出更有效率。

在 PHP 5 中，取决于如何配置 PHP ，有多达五种不同的开始和结束标签可以使用。其中的两种，<?php ?> 和 <script language="php"></script>
总是可用的；从 PHP 5.4.0 开始，短输出标签 <?= ?> 也总是可用的；另外两种分别是短标签和 ASP 风格的标签，虽然有人认为它们更加的简便，但移植性
较差，通常不推荐使用。

注意
    还要注意，如果要将 PHP 嵌入到 XML 或 XHTML 中，则需要使用 <?php ?> 标签以保持与标准兼容。

PHP 7 移除了对 ASP 风格的标签和 <script language="php"> 标签。同样的，在编写 PHP 代码时，为了最大化兼容性，我们建议只使用 <?php ?> 和
<?= ?> 。

只有在 php.ini 配置文件中启用了 short_open_tag 指令或在编译 PHP 时使用了 --enable-short-tags 选项才可以使用短开始标签。
只有在 php.ini 配置文件中启用了 asp_tags 指令才可以使用 ASP 风格的标签，而且它在 PHP 7.0.0 已经被移除了。

注意
    在开发的应用程序或者库可能会被再次分发或者部署在你无法控制的服务器上时，应当避免使用短开始标签，因为在这些目标服务器中很可能就不支持短开
    始标签。

    在 PHP 5.2 和之前的版本中，解析器不允许一个文件的全部内容就只是一个开始标签 <?php 。
    自 PHP 5.3 开始则允许这么做，但在开始标签后要有一个或更多空白字符。

    从 PHP 5.4 开始，短输出标签 <?= 总是可以被识别和有效的，不管 short_open_tag 如何设置。
