当 PHP 解析器碰到 ?> 结束标记时就简单地将其后的内容原样输出（除了紧接着的换行符）直到碰到下一个开始标记；
除非是处于一个条件语句的中间，此时 PHP 解析器会在决定跳过哪些内容之前先去判断条件语句的结果。

对于输出大段文本的情况，跳出 PHP 解析模式通常比将文本通过 echo 或 print 输出更有效率。

在 PHP 5 中，根据配置，有多达五种不同的开始和结束标记可以使用。其中两种，<?php ?> 和 <script language="php"></script> 总是可用的；从 PHP
5.4.0 开始，短输出标记 <?= ?> 也总是可用的；另外两种分别是短标记和 ASP 风格的标记，虽然有人认为它们更加的简便，但移植性较差，通常不推荐使用。

PHP 7 移除了 ASP 风格的标记和 <script language="php"></script> 。同样的，为了编写兼容性最好的 PHP 代码，我们推荐只使用 <?php ?> 和
<?= ?> 。

注意
    如果你要将 PHP 嵌入到 XML 或者 XHTML 中，则需要使用 <?php ?> 标记以遵循标准。

只有在 php.ini 配置文件中启用了 short_open_tag 指令或在编译 PHP 时使用了 --enable-short-tags 选项才可以使用短开始标记。
只有在 php.ini 配置文件中启用了 asp_tags 指令才可以使用 ASP 风格的标记，而且它在 PHP 7.0.0 已经被移除了。

注意
    在开发的应用程序或者库可能会被再次分发或者部署在你无法控制的服务器上时，应当避免使用短开始标记，因为在这些目标服务器中很可能就不支持短开始标记。

    在 PHP 5.2 和之前的版本中，解析器不允许一个文件的全部内容就只是一个开始标记 <?php 。
    自 PHP 5.3 起则允许这么做，但在开始标记后要有一个或更多白空格符。

    从 PHP 5.4 开始，短输出标记 <?= 总是可以被识别和有效的，不论 short_open_tag 如何设置。
