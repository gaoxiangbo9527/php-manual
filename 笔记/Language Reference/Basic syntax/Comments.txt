PHP 支持 C、C++ 以及 Unix shell 风格的注释。
    /* */
    //
    #

单行注释只注释到行尾或者当前 PHP 代码块的结尾，具体要看先碰到哪一个。
这意味着在 // ... ?> 或者 # ... ?> 之后的 HTML 代码将被打印出来，因为 ?> 跳出了 PHP 模式并返回了 HTML 模式，// 或 # 并不能影响到这一点。
如果启用了 asp_tags 配置指令，其行为和 // %> 或 # %> 相同；不过，</script> 标记在单行注释中不会跳出 PHP 模式。

C 风格的注释在碰到第一个 */ 时结束。
要确保不要嵌套 C 风格的注释。
试图注释掉一大块代码时很容易出现该错误。
