当 PHP 解析一个文件时，它会查找开始标签和结束标签，即 <?php 和 ?> ，它们会告诉 PHP 开始和停止解析它们中间的代码。以这种方式进行解析能够让
PHP 嵌入到各种不同的文档中，因为在一对开始和结束标签以外的任何事物都会被 PHP 解析器忽略。

PHP 也包含了一个短输出标签 <?= ，它是更详细的 <?php echo 的简写。

PHP 也允许短开始标签 <? （不推荐使用它，因为只有在 php.ini 配置文件中启用了 short_open_tag 指令或在编译 PHP 时使用了
--enable-short-tags 选项才可以使用短开始标记）。

如果一个文件只包含 PHP 代码，则推荐省略该 PHP 文件尾部的结束标签。这可以防止在结束标签之后意外的添加空白字符或者新的行，因为这样做 PHP 会开
启输出缓冲，从而可能引起不期望的影响，即使编程者本意并不打算此刻在脚本中发送任何的输出。
