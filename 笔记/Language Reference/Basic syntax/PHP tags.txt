结束标记

<?php
?>

PHP 也允许使用短开始标记 <? ，但不推荐使用（只有在 php.ini 配置文件中启用了 short_open_tag 指令或在编译 PHP 时使用了 --enable-short-tags
选项才可以使用短开始标记）。

如果一个文件是纯 PHP 代码，则推荐省略该文件尾部的结束标记。这可以防止在结束标记之后意外的添加空白字符或者新的行，因为这样做 PHP 会开启输出缓冲，从而
可能引起不期望的影响，虽然编程者本意并不打算此刻在脚本中发送任何的输出。

PHP 7.0.0 移除了 ASP 风格的标签 <% 、%> 、<%= 以及脚本（风格的）标签 <script language="php"> 。

PHP 5.4.0 开始，不论 short_open_tag 配置指令是何值，<?= 标记总是可用的。
