同一个文件中可以声明多个命名空间。有两种允许的语法。
    简单的的合并语法。
        namespace MyProject;
            ...

        namespace AnotherProject;
            ...
        不建议使用这种语法。
    花括号式语法。
        namespace MyProject {
            ...
        }

        namespace AnotherProject {
            ...
        }
        推荐使用这种语法。
        除了开始的 declare 语句以外，没有 PHP 代码可以存在于命名空间的花括号之外。
    （个人补充：这两种用法不可以混用）
这种用法主要用途是将多个 PHP 脚本合并到同一个文件中。

为了将全局的非命名空间代码与命名空间的代码合并，只有花括号式语法是被支持的。全局的代码应该被封装在没有命名空间名称的命名空间语句中，如中所示：
    namespace MyProject {
        ...
    }

    namespace { // 全局的代码
        ...
    }

作为一个编程实践，强烈建议不要将多个命名空间合并到同一个文件中。
