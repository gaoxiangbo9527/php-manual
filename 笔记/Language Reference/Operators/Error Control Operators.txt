PHP 支持一个错误控制操作符：即 @ 符号。当将其放置在一个 PHP 表达式前面时，该表达式可能产生的任何错误信息都会被忽略掉。

当触发错误的调用的前面加上了 @ 的情况下，如果你已经使用 set_error_handler() 设置了一个自定义的错误处理程序，那么该错误处理程序仍然会被
调用。虽然这个错误处理程序中可以（而且应当）调用 error_reporting() ，但是得到的结果会是 0 。

如果启用了 trace_error 特性，那么由表达式生成的任何错误消息都会被保存进变量 $php_errormsg 中。每一个错误都会覆盖该变量，所以如果你想使
用它，请尽早检查。

注意：@ 操作符仅对表达式有效。一个简单的经验法则是：如果你可以取到某些东西的值，那么就可以在它前面加上 @ 操作符。比如，你可以将其放置到变量
、函数调用、include 调用和常量的前面，等等。你不能把它放到函数定义和类定义，或者条件结构诸如 if 或 foreach 的前面，等等。

警告：目前，@ 错误控制操作符前缀甚至会禁止报告那些会终止脚本执行的关键错误。除此之外，这意味着假如你使用 @ 去抑制来自一个特定函数的错误，而
且该函数要么不可用，要么拼写错误了，那么脚本会在那里停止，但是却不会告知为什么会停止。
