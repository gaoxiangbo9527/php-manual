instanceof 被用于检测是否一个 PHP 变量是某个特定的类的一个实例化对象。

instanceof 也可以被用于检测是否一个变量是某个继承自某个父类的类的一个实例化对象。

要检测某个对象是否不是某个类的 instanceof ，可以使用 逻辑非操作符 。

最后，instanceof 还可以用于检测一个变量是否是某个实现某个接口的类的实例化对象。

虽然 instanceof 通常与一个类名字面量一起使用，但它也可以与另一个对象或字符串变量一起使用。

如果被测试的变量不是一个对象，instanceof 不会抛出任何错误，它会简单的返回 FALSE 。但是不允许使用常量。
个人测试：从 PHP 7.3.0 开始，可以使用常量了，但是因为常量不能保存对象，所以肯定是返回 FALSE 。

有几个陷阱需要注意。在 PHP 5.1.0 版本以前，如果类名不存在，那么 instanceof 将自动的调用 __autoload() 。此外，如果类没有被加载，将出现
致命的错误。这问题可以通过使用一个动态的类引用或一个包含类名的字符串变量来解决。

instanceof 操作符是在 PHP 5 中被引入的。在这之前使用的是 is_a()，但是 is_a() 已经被弃用，取而代之的是 instanceof 。注意，从 PHP
5.3.0 开始，is_a() 不再被弃用。

另请参阅 get_class() 和 is_a() 。
