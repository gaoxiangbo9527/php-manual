许多编写面向对象应用程序的开发人员会为每个类的定义创建一个 PHP 源文件。最大的烦恼之一就是必须在每个脚本（每个类一个）的开头写一个需要包含的长
的列表。

在 PHP 5 中，这不再是必须的了。spl_autoload_register() 会注册任意数量的自动加载器，让类和接口可以自动的被加载，如果当前它们还没定义的话。
通过注册自动加载器，PHP 在失败和报错之前提供了最后一次机会去加载类或接口。

提示 虽然 __autoload() 函数也能被用于自动加载类和接口，但最好使用 spl_autoload_register() 函数。这是因为它是一种更灵活的选择（允许在应
用程序中指定任意数量的自动加载器，比如在第三方库中)。因为这个原因，所以不推荐使用 __autoload() ，而且从 PHP 7.2.0 起它已经被废弃了。

注意：
    在 PHP 5.3 之前，__autoload() 函数中抛出的异常不能被 catch 块捕获并且会导致一个致命的错误。从 PHP 5.3 开始，这是可能的，前提是抛出
    的是一个自定义异常，然后该自定义异常才是可用的。__autoload() 函数可以被用于去递归地自动加载该自定义异常类。

注意
    如果在 CLI 的交互模式下使用 PHP ，自动加载是不可用的。

注意
    如果类名假设被用于 call_user_func()，则它可能会包含一些危险的字符，比如 ../ 。建议你在这样的函数中不要使用用户的输入或者在
    __autoload()（个人补充：应该是在自动加载器）中验证下输入。
