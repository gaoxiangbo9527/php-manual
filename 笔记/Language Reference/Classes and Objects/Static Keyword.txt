把类属性或方法声明为静态，就可以不实例化类而直接访问它们。

静态属性不能通过一个类已实例化的对象来访问，但静态方法可以。
（个人补充：意思是静态属性不能由对象通过 -> 操作符访问，但静态方法可以由对象通过 -> 访问；另外静态属性和方法均能由对象通过 :: 操作符来访问）。

为了兼容 PHP 4，如果（声明静态属性和方法时）没有指定可见性，（静态）属性和方法默认为 public。

Static methods
    由于不需要创建实例化对象即可调用静态方法，所以伪变量 $this 在静态方法中不可用。

    警示：在 PHP 5 中，静态化的调用非静态方法会产生一个 E_STRICT 级别的警告。

    注意
        在 PHP 7 中，静态化的调用非静态方法是不建议的并且会产生 E_DEPRECATED 级别的警告。将来可能会移除对静态化的调用非静态方法的支持。

Static properties
    静态属性不可以由对象通过 -> 操作符来访问。

    在PHP 5.6.0 之前，静态属性同其他静态变量一样，只能用字面量或者常量（即常值）初始化，而不能用表达式初始化。
    自PHP 5.6.0 开始，可以使用常量表达式，即只要能在编译时得到其值的表达式就可以。

    从 PHP 5.3.0 开始，能够通过一个变量来引用一个类，但是这个变量的值不能是一个关键字，比如 self 、parent 和 static 。
