对象接口能够让你编写代码去指定哪些方法某个类必须实现，却又不用指明如何实现这些方法。

接口使用同类一样的方式定义，但是使用的是 interface 关键字，而且其中定义所有的方法都必须是空的（即没有花括号部分）。
（个人补充）接口中的方法不用加abstract关键字。

接口中定义的所有方法都必须是公有的，这是接口的本质。

注意，接口中是可以声明构造方法的，这种用法在某些情况下是很有用的。

implements
    要实现一个接口，需要使用 implements 操作符。

    实现接口的类，必须实现接口中定义的所有方法，否则会报一个致命错误。

    实现接口的类，必须使用同接口中完全一致的签名来定义方法，否者会一个报致命错误。

    如果需要的话，类可以实现多个接口，用逗号来分隔多个接口的名称。
    (个人补充)，接口可以多继承，也是用逗号来风格多个接口的名称。

    接口可以像类一样使用 extends 关键字进行继承。

    PHP 5.3.9 以前，一个类不能实现两个包含了同名方法的接口，因为这么做会引起歧义；
    PHP 5.3.9 以后可以这么做，前提是这些同名的方法的签名必须完全一致。

Constants
    接口中可以定义常量。
    接口种的常量和类常量的用法完全相同，但是却不能被子类或子接口覆盖。

接口加上类型提示，提供了一种很好的方式来确保某个对象包含有某些方法。
