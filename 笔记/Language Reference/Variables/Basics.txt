PHP 中的变量是由一个美元符号后面跟变量名来表示的。变量名是大小写敏感的。

变量名同 PHP 中的其他标签遵循着相同的规则。一个有效的变量名由字母或者下划线开头，后面跟上任意数量的字母，数字，或者下划线。按一个正则表达式，
它将这样表示：[a-zA-Z_\x7f-\xff][a-zA-Z0-9_\x7f-\xff]* 。

注意：这里所说的字母是 a-z，A-Z，以及 ASCII 码中从 127 到 255（0x7f-0xff）的字符。

注意：$this 是一个特殊的变量，它不能被赋值。

提示：另请参阅 用户空间的命名指南 。

默认情况下，变量总是按值赋值的。也就是说，当你把一个表达式赋值给一个变量时，原表达式的整个值会被复制到目标变量中。例如，这意味着在将一个变量的
值赋值给另一个变量之后，修改这些变量中的一个将不会影响另外一个。

PHP 也支持另外一种赋值给变量的方式：按引用赋值。这意味着新的变量只是简单的引用（换句话说，成为了一个别名或者指向）了原变量。修改新的变量会影
响原变量，反之亦然。
为了按引用赋值，简单地将一个 & 符号加到原变量的前面即可。
一项值得注意的事情是只有命名变量才可以按引用赋值。个人补充：命名变量指的是有名字的变量，更确切的说，只有有名字的值才可以按引用赋值。

在 PHP 中初始化变量不是必须的，但这么做是一种非常好的实践。未初始化的变量根据其使用的上下文，会具有其类型的默认值 —— 布尔型默认为 FALSE，整
型和浮点型默认为 0 ，字符串会设置为一个空字符串（如在 echo 中使用）并且数组会称为一个空数组。
个人补充：其实是从 null 转换为其他类型的。所以对象的话会成为一个空的 stdClass 对象。对于资源的话，PHP 是不支持从其他类型转换为资源的，这里
也一样。

在将一个文件包含到另一个使用相同变量名的文件的情况下，依赖于未初始化变量的默认值会发生问题。使用未初始化的变量会发出 E_NOTICE 级别的错误，但
在向一个未初始化的数组追加元素时不会报该错误。isset() 语言结构可用于检测变量是否已经初始化。
