PHP 为任何它运行的脚本提供了大量预定义变量。由于大多数的预定义变量依赖于所运行的服务器，服务器的版本和设置以及其他因素，因此无法完整的记录它
们。当 PHP 运行在 命令行 上时，其中一些预定义变量将不可使用。有关这些变量的列表，请参阅 保留的预定义变量 的章节。

PHP 还提供了一组额外的预定义数组，其中包含了来自 web 服务器（如果适用的话）、环境以及用户输入的变量。这些数组非常特殊，因为它们是自动全局的
—— 即在每个作用域中都自动可用。因为这个原因，它们经常被称为“超全局变量”（PHP 中没有用户定义超全局变量的机制）。

注意：在 PHP 5.4 以前，用 HTTP_*_VARS 变量这种旧方法而不是超全局变量去检索与 HTTP 请求相关的信息仍然是可行的。从 PHP 5.0.0 开始，通过
register_long_arrays 指令能够禁用这个特性。

注意：可变变量
    在函数或类方法中，超全局变量不能被用作可变变量。

如果 variables_order 中的某些变量没有被设置，那么它们对应的 PHP 预定义数组也将被留空。
