return 会将程序控制返回给调用模块。执行会在调用被调用模块的后面的表达式处恢复。

如果在一个函数中调用 return 语句，该语句会立即结束当前函数的执行，并将其参数作为函数调用的值返回。return 还会结束 eval() 语句或脚本文件的执行。

如果从全局作用域调用 return ，那么会终止 当前 脚本文件的执行。
如果当前脚本是被 include 或者 require 的，那么控制权会返回给该文件的调用文件。
如果当前脚本是被包含的（include 或者 require），那么 return 的值会作为包含调用的值被返回。
如果 return 是在主脚本文件中被调用的，那么脚本的执行会结束。
如果当前脚本文件是 php.ini 中的配置选项 auto_prepend_file 或者 auto_append_file 中命名的，则此脚本文件中止运行。

注意
    既然 return 是一个语言结构并且不是一个函数，所以包围参数的圆括号不是必须的并且也不推荐使用它们。

注意
    如果没有提供参数，那么圆括号必须省略并且会返回 NULL 。使用圆括号调用 return 但不带参数将导致解析错误。

来自用户贡献的笔记
    return 在脚本中被调用时，它会结束 当前 脚本的执行，而不是 所有 脚本 的执行，要重点理解当前这一词的含义。
