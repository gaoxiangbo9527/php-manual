环境变量

描述
    通过环境方式传递给当前脚本的变量的关联数组。

    这些变量从 PHP 解析器所运行的环境中导入到 PHP 的全局命名空间中。

    大多数这样的变量是由 PHP 所运行在的 Shell 提供的，并且不同的系统运行的 Shell 类型也可能不同，所以不可能有一份确定的列表。
    请查看你的 Shell 的文档来获取已定义的环境变量的列表。

    其它环境的变量（即非来自于 Shell 的环境变量）包括 CGI 变量，无论 PHP 是作为服务器模块还是 CGI 处理器运行，都会出现在 $_ENV 中。

    在 PHP 5.4.0 之前，$HTTP_ENV_VARS 中包含着和 $_ENV 相同的初始信息，但它却不是一个超全局变量。
    注意，$HTTP_ENV_VARS 和 $_ENV 是不同的变量，PHP 处理它们的方式也不同。

更新日志
    5.4.0	移除了 register_long_arrays 指令（功能），所以 $HTTP_ENV_VARS 不再有效。
    5.3.0	废弃了使 $HTTP_ENV_VARS 生效的 register_long_arrays 指令。
    4.1.0	引入 $_ENV，弃用 $HTTP_ENV_VARS。

注意事项
    注意：$_ENV 是一个超全局变量，或者说自动全局变量。这就意味着它可以在脚本的任意作用域中使用。不需要进行 global $variable; 就可以在函数
    和方法中访问它。
