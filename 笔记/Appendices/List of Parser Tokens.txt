PHP 语言的各个不同部分在内部由 T_SR 之类的类型表示。在解析错误中 PHP 会输出一个像这样的一个标识符，像
"Parse error: unexpected T_SR, expecting ',' or ';' in script.php on line 10."

你应该已经知道 T_SR 的意思了。对于每个不知道的人来说，这里有一份这些标识符、PHP 语法和手册中对应的引用位置的表格。

注意：使用 T_* 常量
    下方列出的所有令牌也被定义为了 PHP 常量。它们的值是根据 PHP 底层解析器基础结构自动生成的。这意味着令牌的具体值可能会在两个 PHP 版本之间
    发生变化。例如，在 PHP 5.3 中，T_FILE 常量是 365，而在 PHP 5.4 中，相同的值现在指向 T_TRAIT，T_FILE 的值是 369。这意味着你的代码
    不应该直接依赖于从 PHP 版本 X.Y.Z 中获取的原始 T_* 值，以提供跨多个 PHP 版本的兼容性。相反，你的代码应该使用自定义值（使用像 10000
    这样的大数）和一种适用于 PHP 版本和 T_* 值的适当策略。
